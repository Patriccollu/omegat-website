Précisions relatives aux versions

Veuillez noter que les catégories mentionnées ci-dessous ne s'excluent pas mutuellement : une version peut très bien être « standard, avec JRE, pour Windows » ou « récente, avec JRE, pour Linux », etc.

Version standard

La version standard est la version normalement utilisée ; elle se distingue par sa fiabilité. La documentation qui l'accompagne est relativement à jour, du moins en ce qui concerne la documentation en anglais. Elle n'intègre cependant pas toutes les fonctionnalités de la version « récente ».

Version récente

La version récente (ou « Latest ») est plus actuelle que la version standard et intègre de nouvelles fonctionnalités. Globalement, elle est relativement stable, mais elle peut comporter des bogues. D'autre part, sa documentation est moins détaillée, en particulier les informations concernant les nouvelles fonctions. Les utilisateurs qui ont l'habitude d'OmegaT préfèrent souvent la dernière version.

Plateforme (système d'exploitation)

OmegaT se décline en versions spécifiques pour les plateformes Windows, Linux et Mac OS X. Une version indépendante des plateformes est également disponible ; elle fonctionne sur tout système d'exploitation intégrant une version adéquate de l'environnement d'exécution Java.

Avec ou sans JRE

L'environnement d'exécution Java (JRE) doit être installé sur votre ordinateur pour qu'OmegaT puisse fonctionner.

Si un JRE adéquat est déjà installé sur votre ordinateur, vous pouvez télécharger une version d'OmegaT sans JRE – le téléchargement sera plus rapide et l'installation demandera moins d'espace sur le disque dur. Dans ce cas, choisissez par exemple une version « standard » ou « récente » puis sélectionnez la variante portant la mention correspondante (« Java not included », « without_JRE »).

Si vous deviez installer OmegaT avec son JRE, alors qu'un JRE est déjà présent sur votre ordinateur, il n'y aura pas de conflit entre les versions en raison d'installations distinctes. Par conséquent, l'installation d'OmegaT avec le JRE reste la méthode la plus sûre. Si vous n'êtes pas certain d'avoir une version appropriée du JRE, sélectionnez la version d'OmegaT avec JRE (ou la version par défaut).

Note à l'attention des utilisateurs de Mac OS X : le JRE étant installé par défaut sur Mac OS X, la version correspondante d'OmegaT ne comporte pas de JRE.

Note à l'attention des utilisateurs de Linux : la plupart des distributions Linux sont fournies avec un environnement d'exécution Java, cependant certaines implémentations Open Source ne fonctionnent pas toujours correctement avec OmegaT (incompatibilité qui ne s'avère éventuellement pas immédiatement). Nous recommandons aux débutants sur Linux de choisir une version d'OmegaT avec JRE. Les utilisateurs avertis pourront préférer une installation du JRE d'Oracle (anciennement Sun) au niveau système.

Version Web Start

La version Web Start est une solution d'installation simple permettant d'exécuter la version actuelle d'OmegaT.

L'une des particularités de cette version est l'installation automatique des mises à jours.

Durant l'installation, en fonction de votre système d'exploitation, vous recevrez peut-être plusieurs avertissements de sécurité. Le certificat est autosigné « Didier Briel ».

Les autorisations que vous accordez à cette version (par exemple, un « accès illimité à l'ordinateur ») sont identiques aux autorisations que vous donnez à la version locale (c'est-à-dire un accès au disque dur de l'ordinateur).

L'installation de la version Web Start requiert Java 1.3 ou ultérieur sur votre ordinateur.

Engagement de confidentialité : le Java Web Start d'OmegaT ne stocke aucune de vos données sur les serveurs Web d'OmegaT. L'application est exécutée sur votre machine ; vos documents et mémoires de traduction restent sur votre ordinateur et le projet OmegaT n'aura en aucun cas accès à vos travaux ou informations.

Anciennes versions

Ces anciennes versions sont devenues obsolètes et sont ainsi sans intérêt pour la plupart des utilisateurs. Certaines restrictions imposées par le système d'exploitation ou la version Java utilisés pourront toutefois obliger certains utilisateurs à préférer une version plus ancienne.

Version de développement

La version de développement (alpha) reflète la phase de développement la plus récente. Elle n'est recommandée qu'aux développeurs et testeurs assidus. Elle peut également être obtenue via la commande de Subversion :

svn co https://omegat.svn.sourceforge.net/svnroot/omegat/trunk omegat

Paquets code source

Les paquets de code source sont réservés aux développeurs, ils sont sans utilité pour les utilisateurs. Ils sont placés dans les dossiers des versions compilées correspondantes.
