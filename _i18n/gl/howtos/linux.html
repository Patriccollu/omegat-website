Guía de uso de OmegaT en Linux

Esta guía explica como usar OmegaT en Linux. Está especialmente dirixida ás persoas que teñen pouca experiencia no uso de Linux.

Java

OmegaT non funciona de forma adecuada con todas as versións de Java. Se xa ten unha versión de Java instalada no seu sistema (o cal é probable) e quere usar esta instalación que abarca todo o seu sistema de forma conxunta con OmegaT, deberá asegurarse de que é adecuada. Poderá facelo abrindo a terminal (tamén coñecida como consola ou ventá de liña de comandos, exemplos de programas comúns para esta función son o konsole de KDE ou o gnome-terminal de GNOME) e escribindo:

java -version

O sistema daralle información sobre a versión de Java instalada no seu sistema.

Versións de Oracle (anteriormente Sun) de Java: comprobe os requisitos do sistema na documentación de OmegaT (readme.txt) para saber cal é a versión de Oracle Java que precisa (é dicir, se a súa versión de Java é suficiente).

Versións de Java de terceiros: ata fai relativamente pouco tempo, OmegaT só funcionaba coa versión de Java de Oracle/Sun. A situación mudou e agora pódese executar OmegaT coa implementación de Java OpenJDK que inclúen algunhas distribucións de Linux, mais é posible que algunhas funcións non estean dispoñibles (tales como o conxunto completo de opcións da liña de comandos) ou que empregue unha fonte de caracteres distinta. Ademais diso, existen informes de erros, como problemas na forma na que se mostra a ventá ou na función de segmentación.

Se non está seguro de se a súa versión de Java é adecuada, pode seguir estes pasos:

  • Instale unha versión de OmegaT sen Java e comprobe se funciona. Se non funciona, pode probar a instalar unha versión de Java diferente no seu sistema.
  • Instale unha versión de Java distinta á que inclúe o sistema por defecto e execute OmegaT empregando esta versión.
  • Instale a versión de OmegaT que inclúe Java.

Se determina que a versión de Java instalada no seu sistema non é compatible con OmegaT e desexa substituír esa versión por unha de Oracle Java, primeiro deberá comprobar se existe unha paquete de Oracle dispoñible para a súa distribución de Linux (SUSE, Ubuntu, etc.), xa que isto facilitará a instalación.

Se obtén a versión Oracle de Java directamente de Oracle, a instalación debería ser sinxela, pero terá que configurala de forma manual para poder convertela na versión por defecto do sistema. Non recomendamos que se faga isto, sobre todo se non ten experiencia empregando Linux.

A terceira opción, instalar unha versión de OmegaT que inclúe Java, é sempre unha opción \xABsegura\xBB e recomendada.

Seleccionar e descargar unha versión de OmegaT

As seguintes versións de OmegaT están dispoñibles para Linux:

\xABEstándar\xBB, \xABÚltima\xBB, \xAB/troncal\xBB e \xABComezo web\xBB. Recomendamos empregar a versión \xABÚltima\xBB. (Atopará explicacións sobre as diferencias entre as versións na sección \xABDescargas\xBB da páxina web de OmegaT.)

Con ou sen JRE incluído, por exemplo, a versión multiplataforma sen JRE, é dicir, \xABOmegaT_2.1.7_02_Beta_Without_JRE.zip\xBB e a versión específica para Linux con JRE, é dicir, \xABOmegaT_2.1.7_02_Beta_Linux.tar.bz2\xBB. Escolla se precisa unha versión con ou sen JRE tendo en conta a situación do seu sistema con respecto a Java (ten máis información ao respecto arriba).

Ao ser un software de código aberto, moitas distribucións de Linux incluíron OmegaT nos seus repositorios, polo que pode instalar OmegaT a través deles. Non obstante, é preciso que teña en conta que:

  • A maioría das distribucións de Linux empregan a versión OpenJDK de Java por defecto, No momento no que se redacta este texto, OmegaT funciona moito mellor coa versión de Oracle de Java (ten máis información ao respecto arriba).
  • Por norma xeral, os repositorios inclúen a versión \xABEstándar\xBB de OmegaT. É habitual que a versión \xABÚltima\xBB teña máis funcións dispoñibles.

Teña en conta que é posible instalar varias versións de OmegaT en Linux sen que haxa conflitos entre elas, inda que sempre accederán aos mesmos ficheiros de configuración do usuario (a menos que os execute dende unha conta de usuario distinta).

Instalar OmegaT

A versión de Linux de OmegaT inclúe o seu propio instalador, o script linux-install.sh. Este script:

  • coloca os ficheiros de programa de OmegaT en /opt/omegat/OmegaT-<versión>;
  • inclúe scripts de execución de bash e kaptain;
  • crea ligazóns dende /usr/local/bin aos scripts de execución.

O script de bash é un ficheiro moi básico que pode editar de forma manual con opcións de liña de comandos para executar e configurar OmegaT.

O script de kaptain (ou \xABgramática\xBB, tal e como se denomina aos scripts de kaptain) permítelle escoller e seleccionar opcións de execución e configuración a través de unha interface gráfica de usuario fácil de utilizar. Pode escoller o idioma da interface de usuario, asignar máis memoria a OmegaT ou introducir unha contrasinal do Tradutor de Google. Pode personalizar o script de execución de kaptain editando o ficheiro omegat.kaptn cun editor de texto plano para que se empreguen as súas preferencias por defecto. Atopará comentarios de axuda no ficheiro. Teña en conta que para poder executar o script de kaptain primeiro terá que instalar kaptain. Se non está incluído no repositorio da súa distribución, poderá descargalo dende kaptain.sourceforge.net.

O script fai o proceso de instalación máis cómodo, pero se o prefire pode instalar OmegaT de forma manual. So ten que descomprimir o ficheiro no cartafol que desexe. Neste caso non se escribirán os ficheiros de execución no seu camiño de execución.

Executar OmegaT dende a liña de comandos

Inda que non é preciso executar OmegaT dende a liña de comandos, facelo pode ser útil e aporta información de depuración. Para executalo dende a liña de comandos, abra unha ventá de terminal (non precisará permisos de administrador). Navegue ata o cartafol que contén o ficheiro OmegaT.jar, por exemplo:

cd /opt/omegat/OmegaT_2.3.4

e confirme con INTRO.

Despois terá que introducir o comando de execución e confirmar con INTRO. O comando de execución básico é:

java -jar OmegaT.jar

O comando diferencia entre maiúsculas e minúsculas.

Isto non será suficiente se emprega ou desexa empregar unha instalación de Java que non está instalada a nivel do sistema, terá que indicarlle ao sistema onde pode encontrar Java. Se instalou unha versión de OmegaT con JRE, o script de execución que inclúe xa ten a información do camiño que precisa. Se instalou unha versión de Java diferente pero non a estableceu como a versión por defecto a nivel do sistema, deberá incluír o camiño completo ao binario de Java no comando de execución. Debería usar un comando como /usr/lib/jvm/java-6-sun-1.6.0.16/jre/bin/java; durante o proceso de instalación, tome nota do lugar no que está instalado Java para poder atopalo máis tarde. Polo tanto, o comando sería (neste caso)

/usr/lib/jvm/java-6-sun-1.6.0.16/jre/bin/java -jar OmegaT.jar

Non confunda o camiño a OmegaT.jar e o camiño do comando de Java no seu script de execución. O primeiro (co comando de cd) é preciso se o script de execución non está no mesmo cartafol que OmegaT.jar. O segundo é necesario se a versión de Java que desexa usar non é a versión que emprega por defecto o seu sistema.

Executar OmegaT dende un script de execución

Como xa mencionamos, OmegaT inclúe un script de execución de bash. Unha vez o teña copiado ou ligado ao camiño de execución (proceso que se realiza de forma automática se emprega o script de instalación incluído no paquete), poderá executar OmegaT premendo Alt+F2 e, na ventá que se abre, escriba omegat e prema INTRO para confirmar.

O script de execución de bash é un ficheiro executable por defecto, pero é posible que o seu sistema cambie os seus permisos (durante o proceso de desempaquetado do ficheiro) e que teña que volver a facelo executable.

Forma sinxela de executar OmegaT

Pode engadir OmegaT ao seu menú de aplicacións ou crear unha icona no escritorio ou no panel para poder executalo de forma máis sinxela. Hai múltiples posibilidades e múltiples formas de implementalas, por non falar das abundantes distribucións.

Aspecto das fontes en OmegaT en Linux

O aspecto das fontes en OmegaT en Linux pode variar moito, dependendo de:

  • A versión de Java que emprega (unha mesma fonte pode variar moito da versión Sun á versión OpenJDK de Java)
  • As fontes dispoñibles/seleccionadas
  • O tamaño da fonte
  • Se o anti-aliasing está activado (está é unha opción da execución mediante liña de comandos, pero pode activala no script de execución de kaptain)

Desinstalar OmegaT

Para eliminar completamente OmegaT do seu sistema deberá:

Borrar o cartafol do programa de OmegaT. Pode restauralo de forma fácil no futuro.

Eliminar o cartafol /home/<usuario>/.omegat de calquera conta de usuario dende a que se teña executado OmegaT. Isto eliminará calquera ficheiro de configuración que fose creado (como poden ser os cambios ás regras de segmentación) e, se o cartafol foi eliminado da forma \xABadecuada\xBB, é dicir, non movéndoo ao lixo, non poderá restauralo.

Elimine calquera script de execución e calquera entrada personalizada de OmegaT no seu sistema.

Desinstalar OmegaT non borrará ningún proxecto do seu sistema a menos que os tivese gardado no cartafol do programa ou no cartafol de configuración de OmegaT, as cales non son adecuadas para gardar proxectos.
