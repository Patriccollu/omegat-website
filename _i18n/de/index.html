OmegaT ist eine freie, in Java geschriebene Translation Memory-Anwendung. Es ist ein Werkzeug für professionelle Übersetzer. Es übersetzt nicht für Sie! (Software, die das tut, wird "maschinelle Übersetzung" genannt, und Sie werden sich anderswo umsehen müssen.)

Professionelle Funktionen

  • Fuzzy matching
  • Match-Propagierung
  • Gleichzeitige Verarbeitung von Projekten mit mehreren Dateien
  • Gleichzeitige Verwendung mehrerer Translation Memories
  • Benutzerglossare mit Erkennung von flektierten Formen

Arbeiten Sie mit mehr als 30 Dateiformaten

  • Microsoft Word, Excel, Powerpoint
  • XHTML und HTML
  • Open Document Format (LibreOffice, OpenOffice.org)
  • MediaWiki (Wikipedia)
  • Klartext
  • ... und rund 30 andere Dateiformate

Mächtige Werkzeuge

  • Unicode (UTF-8)-Unterstützung: kann mit nicht-lateinischen Alphabeten verwendet werden
  • Unterstützung für Sprachen, die von rechts nach links geschrieben werden
  • Integrierte Rechtschreibprüfung
  • Kompatibel mit anderen Translation Memory-Anwendungen (TMX, TTX, TXML, XLIFF, SDLXLIFF)
  • Schnittstelle zu Google Translate

Das OmegaT-Projekt und Sie

OmegaT ist nicht nur ein Produkt; Es ist ein aufregendes Projekt und Sie können ein Teil davon sein.

OmegaT ist freie Software. Das bedeutet, dass Benutzer wie Sie diese kostenlos herunterladen und verwenden können. Sie können es auch kopieren und an andere weitergeben.

OmegaT ist Open-Source-Software. Das bedeutet, dass Sie, wenn Ihnen danach ist, OmegaT so modifizieren können, dass es Ihren eigenen Anforderungen entspricht. Für Befürworter von Open-Source-Software ist diese Freiheit sogar noch wichtiger als die Befreiung davon, dafür etwas bezahlen zu müssen.

Das OmegaT-Projekt würde von einer größeren Unterstützung von Freiwilligen in fast allen Bereichen profitieren. Obwohl viele Benutzer dies wissen, ist ihre Reaktion oft "Aber ich kann nicht programmieren!". Die Programmierung ist sicherlich die wichtigste Funktion und ohne Programmierung gibt es kein Programm. Aber Programmierung ist nicht die einzige Aufgabe, die erledigt werden muss. Im Folgenden finden Sie eine Liste der Bereiche, für die weitere Freiwillige mehr als willkommen wären!

Dokumentationserstellung

Dokumentation ist sehr wichtig. Viele von Ihnen, die dies lesen, werden Übersetzer sein, die eine Menge Benutzerdokumentationen übersetzt haben. Haben Sie darüber nachgedacht, selbst Benutzerdokumentationen zu schreiben? Es ist ein echter Augenöffner!

Übersetzung

Dank der Arbeit von Übersetzern in der Vergangenheit wurden OmegaTs Benutzeroberfläche und / oder Hilfetexte in mehrere Sprachen übersetzt. Das heißt, es sind nur noch ein paar Tausend übrig! Die Übersetzung von OmegaT in andere Sprachen ist zudem eine fortwährende Aufgabe, da ständig neue Funktionen hinzugefügt werden. Im HowTo Übersetzen von OmegaT finden Sie Einzelheiten darüber, was Übersetzen von OmegaT bedeutet.

Testen

Eines der nützlichsten Dinge, die OmegaT-Benutzer tun können, ist ... OmegaT zu benutzen. Und wenn sie Dinge finden, die nicht funktionieren, oder Mängel in der Dokumentation, uns davon zu erzählen.

Werbung

Haben Sie jemals einen Artikel für das Journal Ihres Berufsverbandes geschrieben? Wenn nicht, warum nicht einen Artikel über die Verwendung von OmegaT schreiben?

Spenden

Wenn Sie OmegaT nützlich gefunden haben und Ihre Dankbarkeit zum Ausdruck bringen möchten, indem Sie eine Spende auf unser Paypal Konto tätigen.

Programmierung (Java)

OmegaT ist in Java geschrieben. Wenn Sie Programmierer sind und Java kennen, lesen Sie in der Dokumentation im Quelltext unter /docs_devel nach, wie Sie beginnen können.

Programmierung (nicht Java)

Wenn Sie in einer anderen Sprache programmieren können, können Sie Code schreiben, um die Kernanwendung zu unterstützen. Ein solcher Hilfscode wurde beispielsweise bereits in Javascript, Groovy und Tcl/Tk geschrieben.

Aber ich kann gar nicht programmieren, höre ich Sie sagen. Nun, haben Sie darüber nachgedacht, zu lernen? Wenn Sie mit Computern im Allgemeinen schon ziemlich vertraut sind und eine einfache Skriptsprache wie Javascript oder Tcl/Tk wählen, könnten Sie sich selbst genug beibringen, um einfache, aber nützliche Skripte innerhalb weniger Wochen schreiben zu können. Fortlaufende berufliche Weiterbildung ist das aktuelle Schlagwort!
