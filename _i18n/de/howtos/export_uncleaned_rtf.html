Dieses HowTo erklärt, wie man eine "unbereinigte RTF"-Datei aus einem OmegaT-Projekt erstellt. Dieses Format kann ein praktisches Austauschformat für Benutzer anderer CAT-Tools sein, insbesondere Trados und Wordfast Classic.

Zu beachtende Punkte:

Diese Code- / Prozedur-Kombination wurde noch nicht gründlich getestet (obwohl erste Berichte vielversprechend sind).

Das Skript ist nur für die Windows-Plattform verfügbar.

Es ermöglicht Übersetzern nur, unbereinigte Dateien für Trados zu erzeugen (d. h. zu liefern). Es ermöglicht ihnen nicht, zweisprachige Dateien ihrer Kunden zu "übersetzen".

1. Fügen Sie Trados-Segmentbegrenzer ein

Um eine zweisprachige OpenOffice.Org-Datei zu erstellen, die die Trados-Segmentbegrenzer enthält, schließen Sie das OmegaT-Projekt, und führen Sie dann das folgende AutoIt-Skript (erfordert Windows) für die Projektdatei des Projekts: project_save.tmx aus:

www.leuce.com/autoit/uncleanifyTMX.zip

2. Zieldateien erstellen

Erstellen Sie die Zieldokumente wie gewohnt und konvertieren Sie die Dateien zu Microsoft Word.

3. Formatierungen auf Trennzeichen anwenden

Führen Sie in Microsoft Word das folgende Makro aus, um die relevanten Formatierungen auf das / die Zieldokument(e) anzuwenden:

Private Sub AddTw4winStyles()
On Error Resume Next
Dim sMark, sInt, sExt As Style
Set sMark = ActiveDocument.Styles.Add(Name:="tw4winMark", _
    Type:=wdStyleTypeCharacter)
With sMark.Font
  .Name = "Courier"
  .Size = 11
  .Hidden = True
  .Subscript = True
  .ColorIndex = wdViolet
End With
Set sInt = ActiveDocument.Styles.Add(Name:="tw4winInternal", _
    Type:=wdStyleTypeCharacter)
sInt.LanguageID = wdNoProofing
With sInt.Font
  .Name = "Courier"
  .Size = 11
  .ColorIndex = wdRed
End With
Set sExt = ActiveDocument.Styles.Add(Name:="tw4winExternal", _
    Type:=wdStyleTypeCharacter)
sExt.LanguageID = wdNoProofing
With sExt.Font
  .Name = "Courier"
  .Size = 11
  .ColorIndex = wdGray50
End With
End Sub
